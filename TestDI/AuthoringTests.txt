==========================================================================
Visual Studio Team System：撰寫和執行測試概觀
==========================================================================

本概觀描述在 Visual Studio Team System 和 
Visual Studio Team Edition for Software Testers 中，撰寫和執行測試的功能。

開啟測試
-------------
若要開啟測試，請開啟包含測試定義的測試專案或測試中繼資料檔 (副檔名為 
.vsmdi 的檔案)。您可在 [方案總管] 中找到測試專案和中繼資料檔。

檢視測試
-------------
若要查看可用的測試，請開啟 [測試檢視] 視窗。或者，如果您已經安裝 
Team Edition for Software Testers，也可以開啟 [測試管理員] 視窗檢視測試。

若要開啟 [測試檢視] 視窗，請按一下 [測試] 功能表，指向 [視窗]，然後再按 
[測試檢視]。若要開啟 [測試管理員] 視窗 (如果您已經
安裝 Team Edition for Software Testers)，請按一下 [測試]，指向 [視窗]，
然後再按 [測試管理員]。

執行測試
-------------
您可以從 [測試檢視] 視窗和 [測試管理員] 視窗執行測試。請參閱＜檢視測試＞
學習如何開啟這些視窗。若要執行 [測試檢視] 視窗中所顯示的一或多個測試，
請先在該視窗選取測試。
若要選取多個測試，請按住 SHIFT 鍵或 CTRL 鍵，同時按一下想要的測試。
然後按一下 [測試檢視] 視窗工具列中的 [執行測試] 按鈕。

如果您已安裝 Visual Studio Team Edition for Software Testers，也可以
使用 [測試管理員] 視窗執行測試。若要在 [測試管理員] 中執行測試，
請核取您要執行的每個測試旁的核取方塊。然後按一下 [測試管理員] 視窗工具列
中的 [執行測試] 按鈕。

檢視測試結果
--------------------
在您執行一個或一系列的測試時，測試回合的結果會顯示在 [測試結果] 視窗中。
執行中的個別測試則會顯示在單獨的一行，讓您可以查看其狀態。
視窗的上半部包含內嵌的狀態列，提供您完成之測試回合的摘要細節。

若要查看特定測試結果的詳細結果，請在[測試結果] 視窗中按兩下測試。
這個動作會開啟視窗，提供特定測試結果的詳細資訊，例如該測試所傳回的
任何特定錯誤訊息。

變更測試的執行方式
-----------------------------------
每次執行一或多個測試時，會使用設定集合決定這些測試的執行方式。這些設定
包含在「測試回合組態」檔中。

以下是您可對測試回合組態檔進行之變更的部分清單：

- 變更每個測試回合的命名配置。
- 變更在其上執行測試的測試控制器，讓您可以遠端執行測試。
- 蒐集所測試之程式碼的程式碼涵蓋範圍資料，讓您可以檢視
  測試所涵蓋的程式碼行。
- 在執行測試前指定其他要部署的檔案。
- 選取不同的主機 (ASP.NET) 執行 ASP.NET 單元測試。
- 為執行測試的測試代理程式設定不同的屬性。
- 在每個測試回合的開始和結束時執行自訂的指令碼，讓您可以
  確實依照每次執行測試時的要求來設定測試環境。
- 設定測試和測試回合的時間限制。
- 設定在測試回合中的瀏覽器混合，和 Web 測試要重複的次數。

根據預設，每當建立新的測試專案，同時會建立測試回合組態檔。
您可以在 [方案總管] 中按兩下這個檔案，然後變更其中的設定，
藉此變更組態檔(測試回合組態檔的副檔名為 .testrunconfig)。

一個方案可以包含多個測試回合組態檔。這些檔案中只有一個稱為「現用」
測試回合組態檔的檔案，會用於決定測試回合目前使用的設定。您可以按一下 
[測試] 功能表上的 [選取現用測試回合組態]，選取現用的測試回合組態。

測試類型
----------
使用 Visual Studio Team Edition for Software Testers，您可建立數個
不同的測試類型：

單元測試：使用單元測試，以 C++、Visual C# 或 Visual Basic 建立程式化  
測試來檢驗原始碼。單元測試會呼叫類別的方法，傳遞適當的參數，並驗證傳
回值是否為您要的值。
有兩種特定的單元測試：
- 資料導向的單元測試，當您將單元測試設定為，可讓資料來源的每個資料列
  重複呼叫時，就會建立這種單元測試。單元測試會將每個資料列的資料當做
  輸入資料使用。
- ASP.NET 單元測試，是指在 ASP.NET Web 應用程式中使用程式碼的單元測試。

Web 測試：Web 測試包含一系列已排序的 HTTP 要求，您可使用 
Microsoft Internet Explorer，將這些要求記錄在瀏覽器工作階段中。您可取得
有關測試要求之網頁或網站的測試報告特定細節，例如特定網頁是否包含特定字串。

負載測試：您可使用負載測試封裝非手動的測試，例如單元測試、Web 測試
和一般測試，然後再利用虛擬使用者同時執行這些測試。在負載下執行這些
測試會以資料表和圖形方式產生測試結果，包括效能和其他計數器。

一般測試：一般測試是在 Visual Studio 中將現有程式包裝為測試之用。
以下為您可轉換為一般測試的測試或程式範例：
- 現有測試，使用處理序結束代碼傳達測試是通過或失敗。
  0 表示通過，其他值都表示失敗。
- 在測試案例期間用於取得特定功能的一般程式。
- 使用特殊 XML 檔 (稱為「摘要結果檔案」) 傳達詳細結果的測試或程式。

手動測試：手動測試會在測試工作是由測試工程師，而非自動化的程式碼
完成時使用。

已排序的測試：使用已排序的測試，依照您指定的順序執行一組測試。
